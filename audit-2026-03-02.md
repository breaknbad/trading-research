## BUGS FOUND & FIXED

### 1. `market_watcher.py` line 319 — wrong column name ⚠️ FIXED
- **Was:** `t.get("entry_price", 0)` 
- **Problem:** Supabase column is `price_usd`, not `entry_price`. Returns 0 for every position → P&L always wrong.
- **Fix:** `t.get("price_usd", t.get("entry_price", 0))` — tries correct column first, falls back.

### 2. `risk_calc.py` lines 165, 218 — wrong column name ⚠️ FIXED
- **Was:** `pos.get("entry_price", 0)`
- **Same issue.** All risk calculations based on entry price return 0 → risk metrics meaningless.
- **Fix:** Same fallback pattern applied to both lines.

### 3. `market_watcher.py` line 94 — missing bot_id filter ⚠️ FIXED
- **Was:** `trades?status=eq.OPEN&select=*` (queries ALL bots' open trades)
- **Problem:** This is how cross-bot data contamination happens. Market watcher pulls every bot's positions, calculates drawdown across the fleet as if it's one book.
- **Fix:** Added `&bot_id=eq.{BOT_ID}` filter.

## CRITICAL STRUCTURAL ISSUE — bot_id=None defaults

### 12 crypto-trading-system functions default to `bot_id=None`:
- `crypto_compliance_enforcer.get_open_positions(bot_id=None)`
- `crypto_compliance_enforcer.check_compliance(bot_id=None)`
- `crypto_fill_verifier.verify_all(bot_id=None)`
- `crypto_kill_switch.check_kill_switch(bot_id=None)`
- `crypto_kill_switch.check_round_breaker(bot_id=None)`
- `crypto_loss_autopsy.get_closed_trades(bot_id=None)`
- `crypto_stop_enforcer.check_all_stops(bot_id=None)`
- `crypto_fleet_var` — queries all positions (by design for fleet VaR, but dangerous if misused)
- `crypto_position_limits` — queries without bot_id filter
- `crypto_correlation_guard` — queries without bot_id filter
- `crypto_drawdown_throttle` — queries snapshots without bot_id
- `crypto_rapid_stop` — queries positions without bot_id

**Risk:** If ANY caller forgets to pass `bot_id`, the function queries ALL bots' data. This is the root cause of data contamination.

**Proposed fix:** Change all defaults from `bot_id=None` to auto-detect:
```python
from bot_config import BOT_ID
def get_open_positions(bot_id: str = BOT_ID) -> list:
```
This makes the safe path the default path. Callers can still override for fleet-wide checks (like fleet VaR).

**Fleet impact:** This fix should be applied by EVERY bot to their own copies. Proposing this at the meeting.
